# SBTI 首页导流区改版设计

日期：2026-04-10

## 目标

在保留 `SBTI` 测试入口为首屏主角的前提下，重做首页首屏下方的信息区，使其从普通署名列表升级为更高级的 `CT 程序定制工作室` 导流区。

本次改版必须满足以下要求：

- 保留原作者署名，只展示 `原作者：B站@蛆肉儿串儿`
- 移除当前首页里的 `托管` 与 `域名` 两行
- 增加 `CT 程序定制工作室` 品牌展示与 `ctikki.com` 官网入口
- 增加两个产品导流入口
- `AI 八字算命` 链接到 `https://bazi.ctikki.com`
- `AI 简历制作` 链接到 `https://resume.ctikki.com`
- 增加 `产品福利交流群` 二维码展示区
- 不改变原有测试题、计算逻辑、结果页逻辑

## 推荐方案

采用“礼宾式导流区”方案。

首页结构保持三层视觉层级：

1. 第一层：`SBTI` 大标题与 `开始测试` 主按钮，继续作为首屏最强焦点
2. 第二层：原作者署名，作为简洁可信来源说明
3. 第三层：工作室导流区，包含官网入口、两个产品卡片和社群二维码

这样可以保证用户仍然优先看到测试入口，同时把工作室信息整理成一块更像品牌名片的内容区，而不是零散文字列表。

## 页面结构

首页 `#intro` 区域调整为以下结构：

- 主标题
- 开始测试按钮
- 原作者署名行
- 工作室导流容器
  - 左侧：工作室标题、简介、官网按钮、产品入口列表
  - 右侧：交流群二维码卡片

桌面端使用左右分栏布局，移动端退化为纵向堆叠布局。

## 视觉设计

延续当前页面的浅绿自然系基调，但提升为更精致的品牌信息区：

- 使用更明确的卡片分层，而不是简单的三行文字
- 工作室区域使用柔和渐变、细边框和轻阴影，体现“高级工作室名片”气质
- 官网按钮与产品按钮采用统一按钮语言，形成清晰 CTA
- 产品入口卡片采用横向信息条结构，左侧是名称和一句简述，右侧是 `立即前往`
- 二维码区域单独作为展示卡片，视觉上更稳，不与产品按钮混在一起

## 文案设计

首页文案调整为：

- 原作者：`原作者：B站@蛆肉儿串儿`
- 工作室标题：`CT 程序定制工作室`
- 工作室说明：强调 AI 工具、落地页、产品原型、效率工具等定制开发能力
- 官网按钮：`前往官网`
- 产品 1：`AI 八字算命`
- 产品 2：`AI 简历制作`
- 产品按钮统一为：`立即前往`
- 社群标题：`产品福利交流群`

文案基调要克制、专业、偏精品工作室，不写成夸张广告语。

## 资源处理

用户提供的二维码图片当前位于仓库外部路径。为保证部署稳定，实施时需要将该图片复制进仓库资源目录，并通过相对路径在页面中引用。

推荐路径：

- `image/community-group-qr.jpg`

如果图片文件名需要更安全的部署兼容性，应使用纯 ASCII 文件名。

## 实施边界

本次只修改首页首屏静态展示区，不改以下部分：

- 题库数据
- 分数统计逻辑
- 结果页展示
- 图片结果资源
- 提交、返回、重置等交互逻辑

## 错误处理

- 如果二维码图片复制失败，则不要上线引用仓库外绝对路径，避免线上失效
- 所有外链使用完整 `https://` 地址
- 新增按钮应使用 `target="_blank"` 与 `rel="noreferrer"`，避免用户离开当前测试页面
- 移动端必须检查按钮换行、二维码宽度和卡片堆叠效果

## 测试与验证

需要验证：

- 首页能正常显示主标题、开始测试按钮和新增导流区
- `原作者` 仍然存在，`托管` 与 `域名` 两行已移除
- 官网按钮跳转到 `https://ctikki.com`
- `AI 八字算命` 按钮跳转到 `https://bazi.ctikki.com`
- `AI 简历制作` 按钮跳转到 `https://resume.ctikki.com`
- 二维码图片能够正常加载
- 点击 `开始测试` 后测试流程不受影响
- 桌面端与移动端布局都保持可用
