# SBTI 静态完成数与人气排行设计

日期：2026-04-10

## 目标

在首页首屏新增一块静态展示区，用来强化“这个测试已经有人在玩”的社交氛围，并补充一个可手动维护的人气排行。

本次只做静态/手动配置版，不接后端，不做真实实时统计。

## 范围

本次新增两个展示内容：

1. 首页首屏主卡片内，显示累计完成测试人数
2. 首页首屏下方，新增 `人气排行` 卡片

展示位置约束如下：

- `已有 12,631 人完成测试` 放在主标题说明区域内，位于按钮上方
- `人气排行` 卡片放在 `版本升级 / 我们新增了什么` 卡片下面

## 实现方式

采用纯前端静态配置方案。

在 `index.html` 中新增一个前端配置对象，例如：

- `completedCount: 12631`
- `popularRankings: [...]`

页面加载时直接渲染，不依赖接口请求，不引入额外 JSON 文件。

## 首页结构

首页从上到下将变为：

1. 标题与开始测试按钮主卡片
2. `版本升级 / 我们新增了什么`
3. `人气排行`
4. 工作室宣传区

其中：

- `版本升级` 继续强调这是增强版
- `人气排行` 强调“大家最容易测出的热门人格”
- 工作室宣传区继续保留导流作用

## 展示形式

### 累计完成数

采用一行醒目但不抢主标题的文字，置于首屏按钮上方，增强“这个测试已经很多人玩过”的从众感。

### 人气排行

使用单独卡片展示，建议包含：

- 卡片标题
- 右上角小标签：`12,631 人已测`
- 前十榜单列表

每一项建议包含：

- 排名
- 人格 code + 中文名
- 一句短标签
- 百分比
- 进度条

整体视觉应延续现有浅绿白卡风格，但更像“热门榜单”，避免写成数据后台。

## 约束

- 只做静态展示，不假装实时刷新
- 不新增网络请求
- 不影响测试流程、结果计算、图片加载逻辑
- 排行内容后续应便于手工修改

## 验证标准

- 首页可见 `已有 12,631 人完成测试`
- `人气排行` 卡片出现在 `版本升级` 卡片下方
- 榜单显示前十项，且至少包含人格名、短句、百分比、进度条
- 首页其余现有功能不受影响
